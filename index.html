PROJECTS

  • Wellness Tracker API

    View on Github

    The Wellness Tracker API is a RESTful backend service I developed to serve as the core of a personal wellness application. It functions as a digital journal, allowing users to securely track and manage their habits, workouts, and recipes.

    This project was built to deepen my understanding of backend development, focusing on the entire lifecycle from database design and API creation to secure authentication and cloud deployment.

    Key Features

    • User Authentication: Secure user registration and login using JWT (JSON Web Tokens).
    • Data Management: Implemented CRUD (Create, Read, Update, Delete) endpoints for managing user-specific data, ensuring privacy and data integrity.
    • Scalable Architecture: Designed with a modular structure using FastAPI routers, making it easy to add new features like workouts and recipes.

    Technology Stack

    • Framework: FastAPI
    • Database: PostgreSQL
    • ORM: SQLAlchemy
    • Migration: Alembic
    • Deployment: Render (with a CI/CD pipeline using GitHub Actions)

BIWEEKLY NOTES

  • 14 Sept 2025

    The first week of my backend development journey felt overwhelming. My initial approach was a passive one, simply following instructions without a clear understanding of the "why" behind each tool. This quickly broke down when I faced a persistent database migration issue, which forced me to re-evaluate my learning process.

    This roadblock became a pivotal moment. I switched to an active learning approach, independently researching each tool and how it functions within the larger system. This shift was the key to moving forward and solidified my foundational understanding of backend concepts.

    Several core backend concepts that I have learnt:

    • API and Frameworks: I learned that an API is a communication bridge, and a framework like FastAPI simplifies the creation of the endpoints that allow different services to talk to each other.
    • Database Management: I now understand the distinct roles of SQLAlchemy (as an ORM for object-to-database mapping) and Alembic (as a tool for managing database schema changes and history). This provided a crucial "aha!" moment, clarifying why a migrations tool is so essential for maintaining a live database.
    • Security & Best Practices: I implemented secure practices by hashing passwords with passlib and storing sensitive information like the database URL and a SECRET_KEY in .env files, which are crucial for managing credentials securely.
    • Deployment Workflow: I gained hands-on experience with the complete development cycle, from writing code and a test suite with Pytest to setting up a continuous deployment pipeline with GitHub Actions and hosting the application on Render.
